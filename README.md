# bookish-system

How do I interpret user needs and implement them into a program? How does creating "user stories" help with this?
I figure out user needs by listening to what the client says and thinking about what each person needs the system to do. User stories help because they describe what a user wants in a simple way, like "as a user, I want to log in." This keeps the focus on the people using the program. It also breaks the work into small pieces that are easier to build and test.
How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?
I like to start by thinking about who will use the program and what they need before I start building it. I gather the requirements first and then plan things out before writing code. Taking it one step at a time makes it easier to handle. In the future, I want to keep using Agile ideas like working in short sprints and building small parts at a time. I also want to keep doing check-ins and reviews so the team can keep improving.
What does it mean to be a good team member in software development?
A good team member communicates well so everyone knows what they are working on. They do the work they said they would do and help others when needed. They also listen to feedback and stay flexible when things change. Most of all, they work together toward the same goal so the whole project comes together.
